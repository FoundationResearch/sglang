# Milestone: 2026-03-26

## Root cause found and fixed: position-id vs engine-index

**Bug**: sglang's `_compute_internal_swa_extend` and `_run_selection_extend` used `token_positions` (position_ids) for window computation and LMK exclusion. Position_ids have LMK sharing: the LMK at engine index 63 and the real token at engine index 64 both get position_id 63. The LMK exclusion `(pos % page_size == page_size - 1)` incorrectly removed the real token at index 64.

**Fix**: All internal SWA and selection code now uses `engine_indices` (precomputed in `init_forward_metadata` from `extend_prefix_lens + arange(extend_seq_lens)`). LMK exclusion uses `(engine_idx + 1) % page_size == 0` matching the official model.

## Decode upper SWA heads used wrong window

**Bug**: The decode's upper SWA heads used `swa_window_size` from `split_info` (which is the HSA merging window) instead of full causal attention. The official model's upper SWA uses `eager_attention_forward` with a full causal mask (no sliding window). With `window_size=64` and `seqlen=65`, sglang excluded token 0 from the decode SWA attention.

**Fix**: Decode upper SWA heads now use `layer.sliding_window_size` (which is None for HSA layers вҶ’ full causal), matching the official model.

## Confirmed: tilelang vs PyTorch ref is EXACT

Replaced official model's tilelang kernels (`HSA_block_M_group`, `online_topk_group`) with PyTorch reference implementations вҶ’ **max_err = 0.000000**. This proves any remaining error is from sglang's side.

## Efficiency note

Current HSA prefill flow:
1. Save KV (all heads)
2. Dense extend on ALL heads (Triton kernel) вҶҗ includes HSA heads that get overwritten
3. Internal SWA on HSA heads (batched bmm)
4. Selection + sparse attention on HSA heads
5. Overwrite HSA heads in output

Step 2 computes attention for HSA heads (~50% of compute) that is immediately discarded in step 5. This is redundant but correct. The dense extend kernel doesn't support per-head subsetting without conflicting with the KV cache layout (`[Nloc, H_total, D]`), continuous batching metadata, and CUDA graph caching.

**Decision**: Keep the current approach. The 50% redundancy for HSA layers is acceptable. For a model with `full_attn_interleave=2` (every other layer is HSA), this is ~25% extra attention compute overall. A head-aware extend kernel would be the proper optimization, but it's a kernel-level change best done separately.
