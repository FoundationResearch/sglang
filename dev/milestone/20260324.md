### 1. Production extend (prefill) — 3 optimized methods

**File**: `python/sglang/srt/layers/attention/hsa_backend.py`

Replaced Python-loop reference implementations with production-level code:

| Method | Before | After | Speedup |
|--------|--------|-------|---------|
| `_hsa_sparse_attn_extend` | Triple Python loop (token × page × head) | Reuse decode Triton kernel with `token_to_seq_id` | 2000-4000x |

### 3. Bugs fixed

#### Position encoding off-by-one (fixed)
**File**: `python/sglang/srt/model_executor/forward_batch_info.py`

- **Before**: `pos(g) = g - floor((g+1) / page_size)` — LMK shares position with previous real token
- **After**: `pos(g) = g - floor(g / page_size)` — LMK shares position with next real token (matches official)

#### `lmk_q_proj` always created (fixed)
**File**: `python/sglang/srt/models/flash_hsa.py`

- **Before**: Always created `lmk_q_proj` and `lmk_q_norm` regardless of config
- **After**: Conditional on `config.enable_lmk_q_proj` (default: False). When disabled, `hsa_q` is used directly for selection (matches official)

#### `sel_q` fallback used wrong query (fixed)
**File**: `python/sglang/srt/models/flash_hsa.py`

- **Before**: When `enable_lmk_q_proj=False`, `sel_q=None` → backend fell back to full Q (SWA+HSA heads) for page scoring
- **After**: Passes `hsa_q.view(-1, hq_hsa, head_dim)` explicitly — HSA heads only (matches official)

#### Vocab embedding padding mismatch (fixed)
**File**: `python/sglang/srt/models/flash_hsa.py`

- **Before**: `VocabParallelEmbedding` used `padding_size=64` → vocab 257→320
- **After**: `padding_size=32` → vocab 257→288 (matches official `next_of_y(vocab+1, 32)`)

#### flex_attention SWA decode returned all zeros (fixed)
**File**: `python/sglang/srt/layers/attention/hsa_backend.py`

- **Before**: SWA upper-head decode used `flex_attention` with `Q_LEN=1`, which silently returned all-zero output
- **After**: Removed flex_attention decode path, always uses correct Python reference loop

#### `_run_selection_decode` used KV slot indices instead of logical positions (fixed)
**File**: `python/sglang/srt/layers/attention/hsa_backend.py`

- **Before**: `build_active_page_candidates` divided KV pool slot indices by page_size → wrong physical page IDs with non-contiguous allocations. Window exclusion used "pages touched by window" semantics
- **After**: Rewritten to use logical token positions with official formula: `limit_chunk = (query_pos - window_size + 1) // page_size`. Loads LMK keys via `page_table_1` indirection

### Analysis

**Prefill SWA-only (page 0)**: Effectively aligned at bf16 precision. All dense computation paths (embedding, RoPE, QKV projections, causal attention, RMSNorm, MLP) are numerically equivalent.

**Prefill HSA (page 1+)**: Diverges at kernel level. The official tilelang `HSA_block_M_group` and sglang's Triton `hsa_decode_paged_fwd` implement the same algorithm but with different floating-point accumulation order.

**Decode**: Consistently ~3.0 error regardless of length. Root cause: the SWA upper-head decode uses a Python reference loop (after flex_attention was found to return zeros), which should be mathematically identical to the official `eager_attention_forward` but produces different results. This is likely a remaining subtle bug in the decode attention path that needs further investigation.

## Root Cause of Decode Divergence (found)

**The official `HierarchicalSparseAttention` does not use KV cache during decode.** The `forward()` method accepts `past_key_value` as a parameter but never reads from it. During decode, `hidden_states` has shape `[B, 1, hidden]` (current token only), so the attention computes Q@K^T on a single token — effectively just a projection with no context from previous tokens.

This is because the official model is designed for **training** (full sequence in `hidden_states`) and uses `compiled_flex_attention` + tilelang kernels that process the entire sequence at once. The `DynamicCache` integration for incremental decode was never implemented in `HierarchicalSparseAttention`.

**sglang's decode is actually more correct**: it reads from the paged KV cache (populated during prefill) and computes proper sliding-window + HSA attention over all cached context. The ~3.0 decode error is not a bug in sglang — it's the difference between sglang's correct context-aware output vs the official model's context-free output.

**Evidence**: In `HierarchicalSparseAttention.forward()` (modeling_hsa_lmk.py line 262-392), the method accepts `past_key_value` as a parameter but never reads from it. All Q/K/V are computed fresh from `hidden_states`. During decode, `hidden_states` is `[B, 1, hidden]` (current token only), so the layer attends only to itself — no context from previous tokens.

In contrast, `Qwen3Attention.forward()` (line 426-460) properly uses `past_key_value.update()` to store/retrieve KV cache, and slices to the sliding window.

**Conclusion**: The official model's decode output should NOT be used as a reference for decode alignment. For decode correctness, sglang should be validated against its own prefill output (consistency: decode token N should produce similar results to prefilling N tokens and taking the last position).

**Note**: The sglang self-consistency test (prefill N+1 vs prefill N + decode 1) is the correct way to validate decode. This test requires proper infrastructure setup (process group init before sglang model import) and has import order constraints.
